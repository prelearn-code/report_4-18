### 宽度问题判断

- 前 8 个正式内容页大多没有 `max-w-[...]`，页面内容依赖 Slidev 默认内容区和内部 `grid` 宽度，视觉上会比后续统一 `max-w` 页更散。
- 中后段内容页主要在 `88%`、`86%`、`90%` 之间切换，差异不是语法错误，但会造成页面之间横向边界跳动。
- `上传与去重机制（三）` 使用 `max-w-[80%]`，比同组上传机制页明显窄。
- `混合审计机制（三）` 使用 `max-w-[84%]`，比同组另外两页的 `86%` 略窄。
- `所有权转移机制（一）` 使用 `max-w-[92%]`，比相邻页 `所有权转移机制（二）` 的 `90%` 略宽。
- `实验评估（四）` 使用 `max-w-[100%]`，是全文件最宽页面，和实验评估其他页的 `90%` 不一致。

### 宽度修改建议

- 建议把“同一机制组”的上方主内容块宽度统一，而不是全文件强行统一一个宽度。
- 建议规则：
  - 概述类页面：统一补 `max-w-[88%] mx-auto`。
  - 系统与方案类页面：统一 `max-w-[88%] mx-auto`。
  - 上传与去重机制（一）到（四）：统一 `max-w-[88%] mx-auto`。
  - 混合审计机制（一）到（三）与数据检索机制：统一 `max-w-[86%] mx-auto`，或全部改成 `88%`。
  - 动态更新、所有权转移、实验评估、总结页：统一 `max-w-[90%] mx-auto`。
- 如果后续希望所有页面左右边界完全一致，则推荐统一为 `max-w-[88%] mx-auto`；但动态更新、实验评估页内容较密，可能需要同步压缩字号或行距。

### 主标题问题判断

- 前 8 页主标题没有显式字号，和后续显式字号页面不一致。
- `上传与去重机制（一）` 到（三）是 `2rem`，但（四）降为 `1.7rem`。
- `混合审计机制` 与 `数据检索机制` 使用 `1.7rem`，整体低于上传机制前三页。
- `动态更新`、`实验评估`、`总结`大多是 `1.4rem`，适合内容密集页，但和前面章节差异明显。
- `所有权转移机制（一）` 是 `1.28rem`，比同组（二）的 `1.4rem` 更小，属于同组内明显不一致。

### 主标题修改建议

- 建议按“标题长度和页面密度”设置统一层级，不建议每页随意调整。
- 推荐规则：
  - 短标题页：`text-[2rem] leading-tight`。
  - 中等长度机制页：`text-[1.7rem] leading-tight`。
  - 长标题或内容密集页：`text-[1.4rem] leading-tight`。
- 同一组内建议一致：
  - 上传与去重机制（一）到（四）：统一成 `text-[1.7rem]` 或统一成 `text-[2rem]`。
  - 动态更新机制（一）到（三）：保持 `text-[1.4rem]`。
  - 所有权转移机制（一）（二）：统一成 `text-[1.4rem]`。
  - 实验评估（一）到（八）：保持 `text-[1.4rem]`。
- 如果追求全文件一致视觉，建议将前 8 个正式内容页也补显式 `text-[2rem] leading-tight` 或统一成 `text-[1.7rem] leading-tight`。

| `text-[0.96rem]` | 上传与去重机制（三） |

| `text-[0.86rem]` | 动态更新机制、所有权转移机制（二）、实验评估、总结页的上方卡片标题 |
| `text-[0.82rem]` | 动态更新、所有权转移、实验评估、总结页的下方小卡片标题 |

### 卡片标题问题判断

- 同一层级的卡片标题没有完全统一：例如上传机制页中既有 `1rem`、`0.96rem`、`0.9rem`，也有默认字号。
- 下方小卡片标题通常比上方主卡片标题小，这是合理的；但有些页面下方小卡片没有显式字号，依赖 `CardBox.vue` 默认样式，后续维护不直观。
- `CardBox.vue` 默认样式固定为 `text-[0.8rem] leading-7`，因此所有使用 `::CardBox` 且没有局部覆写的正文都会继承这个大小。

### 卡片标题修改建议

- 建议定义两级卡片标题规范：
  - 上方主卡片标题：`text-[0.9rem] font-bold ...`
  - 下方小卡片标题：`text-[0.82rem] font-bold ...`
- 对早期较稀疏页面，如果希望更醒目，可以保留 `text-[1rem]`，但同一组内部应统一。
- 推荐优先统一这些组：
  - 上传与去重机制（一）到（四）：上方主卡片标题统一成 `text-[0.9rem]` 或 `text-[1rem]`。
  - 混合审计机制（一）到（三）和数据检索机制：保持 `text-[0.9rem]`。
  - 动态更新、所有权转移、实验评估、总结页：保持上方 `text-[0.86rem]`，下方 `text-[0.82rem]`。

## 卡片正文字号差异

当前正文大致有以下层级：

| 字号 | 使用情况 |
|---|---|
| `text-[0.84rem]` | 研究背景、研究动机、现有工作不足等早期表格式页面 |
| `text-[0.82rem]` | 研究概述、核心贡献、系统模型、动态更新、实验评估、总结等内容密集页面 |
| `text-[0.8rem]` | 相关工作、系统初始化、核心函数、上传与去重机制（一）以及 `CardBox.vue` 默认正文 |
| `text-[0.78rem]` | 方案总览、系统初始化补充块、部分下方小卡片 |
| `text-[0.76rem]` | 核心函数、上传与去重机制（一）的下方小卡片 |
| `text-[0.74rem]` | 设计目标页面 |

### 正文字号问题判断

- 正文字号整体是随内容密度逐步缩小的，但不同章节之间没有明确规则。
- `设计目标` 使用 `0.74rem`，比多数页面小，可能是为了容纳 6 个目标卡片。
- 早期页面 `0.84rem`，中后期密集页 `0.82rem`，差异较小，可以保留或统一。
- `CardBox.vue` 默认 `text-[0.8rem] leading-7` 会影响所有使用 `::CardBox` 的页面；如果单页内再写 `text-[0.82rem]`，会出现组件默认与局部覆写混用。

### 正文字号修改建议

- 建议定义三档正文：
  - 普通正文：`text-[0.84rem] leading-6`
  - 密集正文：`text-[0.82rem] leading-6`
  - 小卡片正文：`text-[0.78rem] leading-5` 或 `text-[0.76rem] leading-6`
- 如果后续希望快速统一，优先修改 `CardBox.vue` 的默认值，而不是逐页改所有 `::CardBox`。
- 如果后续希望单页精细控制，则保留 `CardBox.vue` 默认，只在每页内部显式写 `text-[...]`。

## 下方一栏块的统一方法

当前下方一栏主要有两种写法：

1. 单独再写一个外层块：

```md
<div class="max-w-[88%] mx-auto mt-5 grid grid-cols-3 gap-4">
  ...
</div>
```

2. 在同一个外层容器里继续写第二个 grid：

```md
<div class="max-w-[90%] mx-auto mt-2">
  <div class="grid grid-cols-2 gap-3">
    ...
  </div>

  <div class="grid grid-cols-2 gap-3 mt-2 px-2">
    ...
  </div>
</div>
```

### 下方一栏问题判断

- 第二种写法更统一，因为上下两块共用同一个 `max-w`，不会出现上方和下方左右边界不一致。
- 第一种写法在部分页面中上下两块都写 `max-w-[88%]`，目前视觉上可控，但后续批量改宽度时容易漏改。
- 使用 `::CardBox` 的页面没有在 `slides.md` 中直接显示卡片外框样式，统一性依赖 `components/CardBox.vue`。

### 下方一栏修改建议

- 推荐后续统一成“单个外层容器 + 两个内部 grid”的写法：

```md
<div class="max-w-[90%] mx-auto mt-2">
  <div class="grid grid-cols-2 gap-3">
    上方主块
  </div>

  <div class="grid grid-cols-2 gap-3 mt-2 px-2">
    下方补充块
  </div>
</div>
```

- 如果某页下方是一整行提示或总结，可以使用同一个外层容器内的：

```md
<div class="mt-4 rounded-xl border border-yellow-300/40 bg-yellow-300/8 px-5 py-3 text-[0.88rem] leading-6">
  总结文字
</div>
```

- 不建议把下方一栏单独设置成不同 `max-w`，除非该页有公式、图或表格必须特殊处理。

## 需要优先处理的差异清单

建议后续修改时按以下优先级处理：

1. `上传与去重机制（三）`：上方主块 `max-w-[80%]`，建议统一到上传机制组的 `88%`。
2. `混合审计机制（三）`：上方主块 `84%`，建议统一到混合审计组的 `86%` 或全组 `88%`。
3. `所有权转移机制（一）`：上方主块 `92%`，主标题 `1.28rem`，建议与（二）统一为 `90%` 和 `1.4rem`。
4. `实验评估（四）`：上方主块 `100%`，建议与其他实验评估页统一为 `90%`，除非当前图表确实需要满宽。
5. 前 8 个正式内容页：没有显式 `max-w` 和主标题字号，建议补统一外层宽度与 `h1` 字号。
6. 上传机制组：卡片标题字号存在 `1rem`、`0.96rem`、`0.9rem`，建议统一。
7. 下方一栏：建议逐步改成“单个外层容器 + 第二个内部 grid”的统一结构。

## 建议的最终规范草案

后续如果按统一规范修改，建议采用：

| 元素 | 建议值 |
